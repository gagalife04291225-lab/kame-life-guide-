## 法規制の扱い（重要）

- CITES I（ホシガメ・マッコード等）→ 診断から除外
- CITES II → 「国内CB飼育可・輸入に書類必要」と明記
- 条件付特定外来（アカミミガメ）→ 掲載するが「新規購入不可」を強調
- 特定外来（ワニガメ・カミツキガメ）→ 診断から除外
- 天然記念物（リュウキュウヤマガメ・沖縄産セマル）→ 診断から除外

## 作業ルール（TeTeさんとの約束）

1. 種数は減らさない。増やす方向で
2. 種名は日本語の通称名を先に出す
3. 英名・専門用語は括弧で補足説明する
4. 製品名・ASINは必ずweb_searchで実在確認してから記述
5. 架空の製品・種名の記述は禁止
6. 専門家1000人レビューを3回以上実施してから完成とする
7. 作業提案前に「今すぐ必要か」を必ず確認する
8. 年数は一切記載しない

## GitHub API操作の鉄則

- ファイル更新：①認証付きGETでSHA取得→②base64エンコード→③PUTで更新
- 全APIコールにAuthorization: token $PAT必須
- Termuxでの作業：/tmp不可→~/を使う
- ヒアドキュメント不可→echoで1行ずつ作成
