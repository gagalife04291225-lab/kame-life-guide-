# Phase2 Method Definition Scope

Version: 1.0
Status: Draft

注意：本文書は Draft である。正式な凍結監査完了の証拠が確認できないため、DESIGN FROZEN 扱いにしない。

本仕様書は、方式定義フェーズ（Method Definition）の範囲を固定するものである。normalize方式そのもの（式・アルゴリズム・順位改善方法）は本仕様書では設計しない。本仕様書が定めるのは「方式定義フェーズで何を決め、何を決めてはいけないか」という境界のみである。Phase1 Stable、Step1、Step2、Ranking Quality Specification v1.0、Variable-A Evaluation Plan v1.0 はすべて確定事項として参照のみ行い、変更しない。

## 1. 方式定義フェーズの目的

方式定義フェーズの目的は、Variable-A Evaluation Plan v1.0 が比較対象として扱う各normalize方式を、評価にかけられる状態まで具体化することである。

目的の限定を明記する。このフェーズは方式を「定義する」が、「優劣を判断する」ことも「採用を決める」こともしない。方式に中身を与えて評価可能な状態にするところで止まり、どれが良いかは後続のEvaluation工程が品質仕様に照らして判断する。方式定義は判断のための対象を用意する工程であって、判断そのものではない。

このフェーズが存在する理由は、Evaluation Plan v1.0 の段階で各方式が「名前の枠」だけの状態（none / effective / hybrid という概念名のみ）であり、実際に測るには中身が要るためである。本フェーズはその中身を確定させる工程の範囲を先に固定する。

## 2. このフェーズで決める事項

方式定義フェーズで決めるのは、次に限る。

- 各方式の中身の確定：none / effective / hybrid、および必要に応じて生じる effective のバリアントについて、それぞれが「何をどう正規化するか」を、評価にかけられる具体性まで確定する。
- 方式の識別名の確定：確定した各方式（バリアントを含む）に、Evaluation工程で一意に扱える名前を与える。バリアントが複数生じる場合、それぞれを独立した方式名として区別する。
- 方式ごとの適用範囲の確定：各方式が、順位決定経路のどの段階に作用し、どの候補集合を対象とするか（品質仕様・Evaluation Planが固定した測定前提と整合する形で）を明示する。
- 方式の前提条件の確定：各方式が成立するために必要な前提（ゼロ割回避などの定義域条件）を、H1〜H6の足切りに引っかからない形で満たしているかを、定義レベルで確認できる状態にする。

これらはいずれも「方式に中身を与える」作業であり、「方式を評価する」作業ではない。

## 3. このフェーズで決めてはいけない事項

次は方式定義フェーズの管轄外とする。踏み込んだ場合はスコープ違反とする。

- 方式の優劣：どの方式が良い・悪いという判断。これはEvaluation工程がプロファイルを取得した後に、品質仕様に照らして行う。
- 採用方式の決定：最終的にどの方式を採るか。これはEvaluation後の採用判断工程の管轄。
- 評価基準の再定義：品質の3成分、改善・悪化の定義、H1〜H6、ゴールデン18の役割、判定手順、解釈規約。これらは確定事項であり、方式定義フェーズは参照するのみで書き換えない。
- 比較手順の変更：Evaluation Plan v1.0 が定めた固定条件・手順・終了条件。方式定義はこれに従う側であり、これを変えてはならない。
- タイブレーク（Variable-B）：同点時の順位決定方法。Variable-Aの全工程を通じて対象外であり、方式定義でも一切扱わない。
- 順位改善方法の目的化：「この方式で順位をこう良くする」という改善目標の設定。方式定義は方式に中身を与えるだけで、その方式が何を改善するかを事前に決めない。改善の有無はEvaluation工程が観測する。

原則：方式定義フェーズは「中身を与える」ことに徹し、「その中身が良いか」「それを採るか」には触れない。前者は定義、後者は評価と判断であり、工程が異なる。

## 4. 入力（前工程から受け取るもの）

方式定義フェーズが前工程から受け取るものは次のとおり。

- Ranking Quality Specification v1.0（DESIGN FROZEN）：方式が満たすべき測定前提（3成分の観測対象、H1〜H6の足切り、同点判定の粒度など）の参照元。方式は、この仕様が定める物差しにかけられる形で定義されなければならない。
- Variable-A Evaluation Plan v1.0（DESIGN FROZEN）：方式が従うべき比較条件・固定条件・比較対象の枠の参照元。特に「対象がいくつでも同一手順で測る」という枠と、比較対象として列挙された方式名（none / effective / hybrid）を受け取る。
- Step2 確定事項：normalize挿入位置に関する原則、同点判定の丸め粒度など、方式定義が整合すべき既確定の測定前提。
- Phase1 Stable：none方式が一致すべき基準点。

これらはすべて確定事項であり、方式定義フェーズはこれらを制約として受け取る。制約を満たす方式のみが、次工程で評価にかけられる。

## 5. 出力（Evaluation Planへ渡す成果物）

方式定義フェーズがEvaluation工程へ渡す成果物は次のとおり。

- 評価可能な状態の方式定義一式：比較対象となる各方式（none / effective / hybrid、および生じたバリアント）について、それぞれが評価にかけられる具体性を持った定義。
- 方式識別名の一覧：Evaluation工程が各方式を一意に扱うための名前の一覧。バリアントを含む。
- 方式ごとの適用範囲・前提条件の記述：各方式が作用する段階・対象集合・成立前提。Evaluation工程の固定条件と照合できる形式。
- 確定事項との整合確認記録：各方式が、品質仕様v1.0・Evaluation Plan v1.0・Step2の測定前提と矛盾しないことを、定義レベルで確認した記録。

成果物の要件：ここで渡すのは「定義」であって「評価結果」ではない。優劣・順位・採用可否は含まない。Evaluation工程が同一手順で公平に測れるだけの、中身の揃った方式群を渡すことが本フェーズの成果である。

## 6. 完了条件（Method Definitionが完了したと言える条件）

方式定義フェーズが完了したと言えるのは、次のすべてを満たしたときとする。

- Evaluation Plan v1.0 が列挙する比較対象（none / effective、必要時 hybrid、および定義過程で生じた全バリアント）のそれぞれについて、評価可能な状態の定義が確定していること。
- 各方式に、Evaluation工程で一意に扱える識別名が与えられていること。
- 各方式が、品質仕様v1.0・Evaluation Plan v1.0・Step2 の確定した測定前提と矛盾しないことが、定義レベルで確認されていること。
- none方式が Phase1 Stable と一致することが定義レベルで確認されていること（基準線の妥当性担保）。
- 出力成果物一式が、Evaluation工程へそのまま引き渡せる形で揃っていること。

完了条件の限定：本フェーズは「全方式の定義が揃った時点」で完了する。「どの方式が良いか」「どれを採るか」が決まることは完了条件に含めない。それらは後続工程の管轄であり、本フェーズの完了はそれに先行する。

## 7. 次工程との接続（Variable-A Evaluationとの関係）

方式定義フェーズは、Phase2の工程構造において Evaluation Plan と Evaluation Results の間に位置する。

Variable-A Evaluation Plan（測り方の確定・DESIGN FROZEN）
→ Method Definition（各方式の中身を確定）← 本フェーズ
→ Variable-A Evaluation 実施（全方式のプロファイル取得・引き渡し）

接続の関係を明記する。Evaluation Plan v1.0 は「どう測るか」を確定済みであり、本フェーズはその測定対象となる「方式の中身」を用意する。本フェーズの出力（評価可能な方式定義一式）が、Evaluation工程の入力となる。

本フェーズはEvaluation Planを書き換えない。Evaluation Planが定めた固定条件・手順に方式が従える形で定義を用意する、という一方向の従属関係である。方式定義がEvaluation Planの手順を変えることはなく、逆にEvaluation Planの制約が方式定義を縛る。

本フェーズ完了後、Evaluation工程は本フェーズの出力を受け取り、Evaluation Plan v1.0 の手順に沿って各方式を品質仕様v1.0にかけ、プロファイルを取得する段階へ進む。
