# Phase2 Variable-A Evaluation Plan

Version: 1.0
Status: DESIGN FROZEN

本計画書は、normalize方式を公平に比較するための実験手順のみを定める。方式の中身（式・実装）、優劣、採用可否には一切触れない。タイブレーク（Variable-B）は本計画の対象外とする。Ranking Quality Specification v1.0（DESIGN FROZEN）、Phase1 Stable、Step1、Step2 はすべて確定事項として参照のみ行い、変更しない。

## 1. 実験の目的

本実験の目的は、複数のnormalize方式を、Ranking Quality Specification v1.0 という共通の物差しにかけ、各方式の品質プロファイルを同一条件で取得することである。

目的の限定を明記する。本実験は「どの方式が優れているか」を決めない。それは本実験が出力するプロファイル群を受け取る後続の判断工程が、当初目的（ランキング品質を改善しながらPhase1 Stableより悪化しないこと）に照らして行う。本実験は判断の材料を、公平かつ再現可能な形で揃えるところで止まる。

## 2. 比較対象

比較対象は次のnormalize方式とする。本節では方式名の枠のみを定義し、各方式の中身は定義しない。

- none：候補間normalizeを適用しない方式。Phase1 Stableと同一の順位決定経路に相当し、全方式共通の基準線を兼ねる。
- effective：候補間normalizeを適用する方式。中身は本計画では定義せず、後続の方式定義工程で確定される。
- hybrid：必要時にのみ比較対象に加える方式。中身は本計画では定義しない。

方式追加の扱い：本実験は上記を標準対象とする。方式定義工程で effective が単一定義でなく複数バリアントを持つ形で確定した場合、それらはそれぞれ独立した比較対象として本計画の同一手順にかける。バリアントの区別は方式名の区別として扱い、手順は変えない。本計画は「対象がいくつであっても同一手順で測る」ことを保証する枠であり、対象数に依存しない。

## 3. 比較条件（全方式で固定すべき条件）

公平性は、方式以外のすべての条件を全方式で同一に固定することで担保する。次を固定条件とする。

- 入力集合：ゴールデン18ケースのみを用いる。ケースの内容・順序・役割分担（正解型／許容型／堅牢性検査型）は Step1 のまま変更しない。全方式に同一の18ケースを与える。
- 基準点：Phase1 Stable を全方式共通の基準線とする。各方式のプロファイルは Stable との差分として読めるようにする。
- 足切り基準：H1〜H6（および該当する堅牢性条件）を全方式に同一適用する。基準を方式ごとに緩めない・厳しくしない。
- 観測対象：品質仕様第1章の3成分（妥当性・決定力・安定性）を全方式で同一に観測する。方式ごとに観測項目を増減しない。
- 判定手順：品質仕様第7章の4段階手順を全方式に同一適用する。
- 解釈拠り所：品質仕様7.7 解釈一貫性規約を全方式に同一適用する。方式名を解釈の材料にしない（7.7.5）。
- 同点判定の粒度・その他の測定前提：Step2 で確定済みの測定前提（同点判定の丸め粒度等）を全方式で同一に用いる。本計画では新たな測定前提を導入しない。

固定条件の原則：本実験で方式間に差が出てよいのは「normalize方式そのもの」だけである。それ以外に方式間で異なる条件が一つでもあれば、その差が交絡となり比較が無効になる。上記固定条件は、この交絡を排除するための最小集合である。

## 4. 実験手順（評価の順番）

方式の評価は、品質仕様第7章の4段階を、方式ごとに同一順序で実施する。方式をまたぐ実施順序は次のとおり。

手順1：基準線の確定
none（＝Phase1 Stable相当）を最初に品質仕様第7章の手順にかけ、そのプロファイルを基準線として確定する。以降の全方式はこの基準線との差分で評価される。基準線が最初に固定されることで、後続方式の観測に一貫した参照点が与えられる。

手順2：各方式の評価
none以外の各方式について、次を方式単位で実施する。
- 第1段階（足切り）：H1〜H6抵触を機械判定者が確認。抵触があれば当該方式を「品質比較対象外」として記録し、以降の段階に進めない。
- 第2段階（定量観測）：足切り通過方式について、3成分の観測値をケースごとに機械判定者が抽出。
- 第3段階（定性観測）：許容型ケースの自然さ採点とトレードオフ事例の解釈を、7.7規約に沿って人間判定者が付与。
- 第4段階（プロファイル確定）：上記を品質仕様の整理形式にまとめ、当該方式のプロファイルを確定。

手順の順序原則：方式を評価する順番自体は結果に影響してはならない（7.7.6：日時・順序への非依存）。手順2の方式間の実施順は任意でよいが、各方式には必ず同一の4段階を適用する。効率化のための早期打ち切り（ある方式が足切りで落ちたら残段階を省く）は、足切り段階に限り許容する。定量・定性段階の途中省略は行わない。

## 5. 各方式から取得する成果物

各方式から取得する成果物は、品質仕様が定める構造化プロファイルであり、次を含む。

- 足切り結果：H1〜H6抵触の有無（品質比較対象内／対象外の別）。
- 成分別観測：妥当性・決定力・安定性それぞれの、Stable比の挙動。ケース種別ごとに分けて保持し、単一値に合算しない。
- トレードオフ記録：向上と低下が同時に生じた事例の、どの成分が上がりどの成分が下がったかの個別列挙。
- 人間解釈メモ：許容型ケースの自然さ採点、トレードオフ事例への解釈コメント（7.7に基づく根拠・前例参照を含む）。

成果物の統一要件：全方式について同一区分・同一粒度で取得する。ある方式にのみ有利／不利な項目を追加・省略しない（品質仕様第7章5節の中立性要件を踏襲）。none（基準線）も同一形式で取得する。

## 6. 品質仕様との接続方法

本計画は、品質仕様 v1.0 を評価エンジンとして利用する。接続関係は次のとおり。

- 本計画は「何を・どの順で・どの固定条件下で測るか」という実験の枠を定める。
- 各方式を実際に測る行為（判定者分業・4段階手順・整理形式・解釈規約）は、すべて品質仕様第7章・7.7 に委ねる。本計画はこれを再定義せず、参照するのみ。
- 改善・悪化の意味づけ（第2・3章）、品質の3成分（第1章）、同点解消の位置づけ（第5章）、公平比較の原則（第6章）も、本計画はそのまま適用し書き換えない。

接続の原則：本計画は品質仕様の外枠（実験運用）であり、品質仕様は本計画の内側で動く測定規約である。両者は階層が異なり、重複も矛盾もしない。本計画が品質仕様に加える変更はゼロである。

## 7. 評価終了条件

Variable-A比較が完了したと言えるのは、次のすべてを満たしたときとする。

- 標準対象（none / effective、必要時 hybrid、および方式定義工程で確定した全バリアント）のそれぞれについて、品質仕様第7章の4段階が完了し、プロファイルが確定していること。
- 足切りで対象外となった方式についても、その事実と抵触内容が記録されていること（対象外も「評価済み」に含む）。
- none の基準線プロファイルが確定し、全方式がそれとの差分で読める状態になっていること。
- 全方式のプロファイルが、同一区分・同一粒度で揃っていること（成果物の統一要件の充足）。

終了条件の限定：本実験は「全方式のプロファイルが揃った時点」で完了する。「どの方式を採用するか」が決まることは終了条件に含めない。採用判断は後続工程の管轄であり、本実験の完了はそれに先行する。

## 8. 次工程へ渡す成果物

本実験が次工程（採用判断工程）へ引き渡すものは次の一式とする。

- 全方式の品質プロファイル：足切り通過方式の構造化プロファイル（成分別観測・トレードオフ記録・人間解釈メモ）。
- 対象外方式の記録：足切りで落ちた方式とその抵触内容。
- 基準線プロファイル：none（Phase1 Stable相当）のプロファイル。全方式差分の参照点として同梱する。

引き渡しの原則：本実験は方式に順位を付けず、優劣の結論を添えずに、プロファイル一式をそのまま渡す（品質仕様第7章6節を踏襲）。どの成分をどれだけ重視して採用を決めるかは、当初目的に照らして後続工程が判断する。本実験はその判断材料を、公平・再現可能・同一形式で用意するところまでを担う。

## 工程構造（補足・確定）

Phase2 は次の順序で進む。

Evaluation Plan（測り方の確定・本文書）
→ Method Definition（各normalize方式の中身を確定）
→ Variable-A Evaluation 実施（全方式のプロファイル取得と引き渡し）

本計画は第1段にあたり、方式の中身に依存しないため、後続2段に先行して凍結できる。採用判断およびタイブレーク（Variable-B）は、この3段の後続に位置し、本計画の管轄外である。
