# 診断ツール再設計 — 作業記録（2026-07-14）

現在の診断エンジン（shindan/species.js の手書き score()）を、事実データ＋共通エンジンに作り替えるための設計。**まだ実装していない。** 属性データが数種そろってから実装する。

---

## なぜ作り替えるか

現行エンジンの実測結果（land ルート 5,184通り総当たり）:

- ヘルマンリクガメが **27.8%** で1位（何を答えても4回に1回）
- 21種中 **9種** はどう答えても絶対に1位にならない（ケヅメ含む）
- 全ルート合計 **30種が死んでいる**（ヒメハコヨコクビガメ含む）

原因:
1. score() が種ごとに手書きの足し算。**項が多い種ほど無条件に高得点**（ヘルマン8項 vs ロシア5項）
2. match を厳しくした種ほど損をする（正確に書いた種が罰される）
3. all ルートで57シグナル中47が常に0

---

## 新設計の核

**種は事実だけを持ち、スコアはエンジンが計算する。** 手書き score() を全廃。

### ファイル分離

| ファイル | 内容 | エンジンが読むか |
|---|---|---|
| `species-facts.json` | 事実（サイズ・温度・費用・行動など） | **読む** |
| `species-knowledge.json` | 経験・失敗談・注意点 | 読まない |
| `species-breeding.json` | 繁殖 | 読まない |

エンジンは facts しか読めない。経験をスコアに混ぜる事故が構造的に起きない。

### 編集原則（最重要）

**分からないことは分からないと書く。** 推測で空欄を埋めない。根拠のない数値を診断に使わない。

### provenance（情報源の型。優劣ではない）

official / peer_reviewed / expert_consensus / expert_opinion / keeper_consensus / keeper_observation / estimated / unstudied

S/A/B ランクは持たない。項目によって最強の根拠は違う（CITESは公的資料で飼育経験より強い等）。

### conflict と caution を分離

- `conflict`: 質の高い根拠同士の対立。多数決で潰さない
- `caution`: リスク報告。推奨されていても危険な場合
- デマは conflict に入れず**除外**（例: petpi.jp「スペングラーは青い鱗」= AI生成）

### limitations.missingData

「分かっていないこと」を情報として持つ。種ページに「この種はまだ分かっていないことが多い」と誠実に書ける。

## 検証状況

3種で2回スキーマを壊して直した（v1→v2）。

### v2 で確認できた成果
- `estimated`（推測）25→8件、`unstudied`（研究なし）12→31件。**推測が減り、正直な空欄が増えた**

### スペングラー = 完成（4頭7年の実データ）
通説を実飼育が覆した:
- 食性: 虫が主食・食いつき常に良い（専門医の「動物質＝環境不良説」への反例）
- ハンドリング: 落ち着く・慣れる・寄ってくる（通説は臆病・神経質）
- 活動: 昼行性
- ピンクマウスと虫を分離（虫OK、哺乳類は不要・死亡例あり）

### まだ埋まっていない
- ヒメハコ（2頭2年）: winterStrategy/socialTolerance/breeding 等が未記入
- ケヅメ（飼育なし）: minFloorArea が資料3倍差で確定不能、cost算出不能

### v2 で見つかった残りの穴（v3で直す）
1. `tempTolerance` の narrow/moderate/wide が機能しない → `heatTolerance`/`coldTolerance` 2軸に（スペングラーとケヅメが両方 moderate になる）
2. ライフステージで飼育形態が変わる種（ケヅメ幼体/成体）→ `lifecycle` を該当種のみに
3. Starter Kit を出せない種（ケヅメ成体=市販機材で完結しない）
