## 1. 確認項目（この順で見る）

### ① Search Console（約5分）
- **検索パフォーマンス**（直近7日 vs 前7日）：クリック / 表示回数 / 平均CTR / 平均掲載順位
- **インデックス作成 > ページ**：インデックス済み件数の推移、除外（未登録）の理由別
  - 見る除外理由：「noindex タグにより除外」「重複」「クロール済み - インデックス未登録」「見つかりませんでした(404)」「サーバーエラー(5xx)」
- **サイトマップ**：検出URL数・エラーの有無
- **新規インデックス**：直近で新しくインデックスされたURL（新規記事が入っているか）

### ② GA4（約5分）
- **集客 > トラフィック獲得**：セッションの参照元/メディア（TikTok流入は `utm_source=tiktok` で分離）
- **エンゲージメント > ページとスクリーン**：人気ページ（page_view 上位）
- **診断ファネル**（イベント数）：`shindan_start` → `diagnosis_result_view` → `diagnosis_result_species_cta` / `diagnosis_result_starterkit_cta` → `affiliate_click`
  - 診断開始率 = `shindan_start` ÷ `/shindan/` の page_view
  - 診断完了率 = `diagnosis_result_view` ÷ `shindan_start`
- **affiliate_click**：イベント数（provider = amazon / rakuten 別）
- **スクロール率**：`scroll`(90%) ÷ page_view ※GA4拡張計測がONの場合のみ

### ③ TikTok（約3分）
- 投稿数（本/週）・再生数（合計/平均）・プロフィールクリック数
- ホームページ流入は **GA4側**で `utm_source=tiktok` のセッション数を見る

## 2. レビュー手順（15分の流れ）

1. （3分）GA4：流入元・人気ページ・診断開始率・診断完了率・affiliate_click・スクロール率を前週比で確認
2. （3分）GSC：クリック / CTR / 順位の前週比、インデックス / 除外 / クロールエラーの変化
3. （3分）TikTok：投稿数・再生数・プロフィールクリック・(GA4で)TikTok流入を記録
4. （3分）改善対象の選定 → 次章へ
5. （3分）先週着手ページの効果測定＋今週の記事テーマをキューイング

---

## 3. 改善手順（P0を最大3ページだけ）

**GSC の実測値のみで対象を選ぶ。** 判断は [DECISION_RULE.md](DECISION_RULE.md) に従う。

1. GSC「検索結果 > ページ」で、**表示回数が多いのに CTR が低い / 順位が伸びない** ページを抽出
2. 各ページを Decision Rule で改善タイプに割り当てる：
   - CTR低 → title改善／ 順位11〜20位 → 内部リンク追加／ 順位4〜10位 → description改善／ 順位1〜3位 → 関連記事・面拡張
   - affiliate_click低 → CTA改善／ 診断離脱 → 診断UX確認／ スクロール率低 → 本文・構成見直し
3. **今週着手するのは最大3ページまで**（全部直さない。実値がトリガーを引いた上位だけ）
4. 実装後は翌週以降の GSC で順位 / CTR の変化を確認（効果測定）

---

## 4. 改善履歴の記録方法

スプレッドシート（またはこのフォルダの記録シート）に1行ずつ追記する：

| 日付 | ページURL | トリガー(実測値) | 改善タイプ | Before(順位/CTR) | After(順位/CTR) | 備考 |
|------|-----------|------------------|-----------|------------------|-----------------|------|
| 例）2026-07-21 | /species/japanese-pond-turtle.html | CTR 1.8%・表示 1,200 | title改善 | 順位9.2 / CTR1.8% | (翌週記入) | Phase2-C式 |

- **Before は着手時点の GSC 実測値**、**After は着手の2〜4週後の GSC 実測値**（反映に時間差があるため）
- 記事追加も同じシートに記録（テーマの選定根拠＝GSCのどのクエリか、を必ず書く）

---

## 5. コンテンツ追加（週1〜2本まで）

- テーマは **GSC で検索需要が実在するもののみ**（表示はあるが専用ページが無い / 順位が低いクエリ）
- 着手前に **既存ページの有無を必ず監査**（同テーマがあれば新規作成せず強化＝カニバリ回避）
- 実在商品のみ・価格記載なし・断定表現なし・医療断定なし
- 作成後：sitemap 追記 ＋ 関連ページからの内部リンク ＋ Article/Breadcrumb/FAQ schema
