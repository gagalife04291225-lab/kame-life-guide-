# 改善判断ルール（DECISION RULE）

> KAME LIFE GUIDE 運営ドキュメント｜「どのページを・どう直すか」を実測値で機械的に決める
> 関連: [WEEKLY_REVIEW.md](WEEKLY_REVIEW.md) ・ [KPI.md](KPI.md) ・ [MONTHLY_REVIEW.md](MONTHLY_REVIEW.md) ・ [OPERATIONS_MANUAL.md](OPERATIONS_MANUAL.md)

**原則：推測で直さない。GSC / GA4 の実測値がトリガーを引いたページだけを、上から数件直す。**

---

## 1. 優先度トリアージ（P0 / P1 / P2）

| 優先度 | 条件（実測） | 意味 | 対応ペース |
|--------|--------------|------|-----------|
| **P0** | 表示回数が多いのに CTR が低い ／ 平均順位 11〜20位 | 要改善（伸びしろ大） | 週 最大3ページ |
| **P1** | 平均順位 4〜10位 ／ 低EPCカテゴリのCTA ／ 診断離脱段 | 改善候補 | P0の後・余力で |
| **P2** | 平均順位 1〜3位・高CTR | 好調（面拡張） | 月数件・内部リンクで波及 |

> 「全部直す」はしない。P0 を毎週最大3件が上限。

---

## 2. Decision Tree（改善タイプの決定）

```
そのページの主トリガーは？
│
├─ CTR が低い（表示は多いのにクリックされない）
│      └─► title 改善（複合キーワード化）… Phase2-C 式
│
├─ 平均順位 11〜20位（あと一歩で1ページ目）
│      └─► 内部リンク追加（関連ページ→当該ページ）… Phase2-B / E 式
│
├─ 平均順位 4〜10位（1ページ目だが上位でない）
│      └─► description 改善（CTRで後押し）… Phase2-D 式
│
├─ 平均順位 1〜3位（すでに上位）
│      └─► 関連記事追加・クラスタ内リンクで「面」を広げる
│
├─ affiliate_click が低い（収益ページなのにクリックされない）
│      └─► CTA 改善（文言・位置。既存要素の範囲で）
│
├─ 診断が start→result で離脱している
│      └─► 診断 UX 確認（離脱している段を特定して見直す）
│
└─ スクロール率が低い（本文が読まれず離脱）
       └─► 本文・構成の見直し（冒頭の結論・見出し・可読性）
```

---

## 3. しきい値の扱い（重要）

- 参考初期値：**CTR < 3%**、順位帯 **11〜20 / 4〜10 / 1〜3**。
- ただし **しきい値は運用開始後の GSC 実分布で更新する**（サイト・クエリで最適CTRは変わるため）。
- 仮の数値で固定しない。毎月の [MONTHLY_REVIEW.md](MONTHLY_REVIEW.md) で分布を見て調整する。

---

## 4. 改善タイプ別・具体手順（過去フェーズの手法に接続）

| 改善タイプ | 何をするか | 参照 |
|-----------|-----------|------|
| title改善 | 「○○の飼い方｜KW1・KW2・KW3｜カメライフガイド」形式へ。本文が扱うテーマから複合KWを選ぶ（推測しない）。60字以内・重複0 | Phase2-C |
| 内部リンク追加 | 関連する種/ガイド/比較/病気ページから当該ページへ自然にリンク。孤立を作らない | Phase2-B / E |
| description改善 | 90〜120字・本文整合・重複0・詰め込みなし。CTRを後押し | Phase2-D |
| 関連記事・面拡張 | クラスタ内の相互リンク、未作成の関連テーマを1本追加（カニバリ監査必須） | クラスタ運用 |
| CTA改善 | ボタン文言・配置の見直し。**商品・価格・断定表現は禁止のまま**。新規Product schemaは追加しない | 品質条件 |
| 診断UX確認 | ファネル該当段（start / result / CTA）の離脱を特定 | 診断 |
| 本文・構成見直し | 冒頭に結論、見出しで区切る、モバイル可読性 | UX |

---

## 5. 禁止事項（改善時も守る品質ガード）

- 実在しない商品を扱わない・価格を書かない・順位を誇張しない
- 「絶対」「最強」など断定表現を使わない・医療断定をしない（病気記事は「可能性」「早めに受診」）
- 同一テーマの重複ページを作らない（**検索カニバリゼーションを最優先で回避**）
- 飼育していない種を実体験のように書かない（Experienceは実飼育種のみ）
